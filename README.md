# Uber Eats Merchant Operations Analysis using SQL

## Project Overview

This project analyzes Uber Eats merchant-level data to understand merchant coverage, delivery performance, ratings, pricing tiers, and promotional activity across different locations.

The goal is to generate operational insights that could help improve delivery efficiency and customer satisfaction.

## Tools Used

- SQL
- Python
- Pandas
- SQLite
- Jupyter Notebook

## Dataset

The dataset contains merchant-level Uber Eats information including:

- city
- state
- restaurant name
- delivery time
- review rating
- review count
- price bucket
- promotion usage
- merchant availability

## Analysis Performed

- Merchant coverage analysis across cities and states
- Delivery time performance comparison
- Merchant ratings and review analysis
- Promotion usage analysis
- Merchant availability analysis
- Pricing tier distribution

## Key Insights

- Merchant coverage varies significantly across locations.
- Delivery times differ across states, indicating operational variation.
- Customer ratings help identify high-performing merchants.
- Promotions are used selectively across merchants.
- Merchant availability influences platform accessibility.
